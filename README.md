> WorkBuddy 智能日志工具 —— 记录生活、生成复盘、追踪成长

---

## ✨ 功能概览

| 功能 | 说明 |
|------|------|
| 📝 **日志保存** | 识别日期格式，自动保存日志到年月文件夹，追加到总日志 |
| 📊 **月志生成** | 基于"复盘六问"方法论，自动生成月度复盘报告 |
| 📈 **年报生成** | 四维度深度分析 + 六阶段年度评估 |
| 🔍 **日志搜索** | 关键词搜索 + 日期范围过滤 |
| 📉 **统计分析** | 写作频率、心情趋势、高频关键词分析 |
| ⏰ **智能提醒** | 每日定时提醒、间隔提醒、重要日期提醒 |


## 📖 使用指南

### 📝 日志保存

**触发方式**：输入日期格式 + 内容

支持的日期格式：
- `M.D` → `5.18`、`12.03`
- `YYYY.M.D` → `2026.5.18`
- `YYYY-MM-DD` → `2026-05-18`

**系统会帮你**：
- ✅ 自动识别日期，确定年份和月份
- ✅ 创建对应的文件夹结构
- ✅ 生成分析（开心/充实/感恩/思考/改进）
- ✅ 追加到总日志
- ✅ 自动Git提交（如已安装Git）

---

### 📊 月志生成

**触发方式**：
- "生成5月月志"
- "帮我写个月度复盘"
- "生成月志"

**基于"复盘六问"方法论**：
1. 回顾目标 —— 这个月的目标是什么？
2. 评估结果 —— 达到了预期吗？
3. 分析原因（正向）—— 超预期的地方在哪？
4. 分析原因（负向）—— 不符合预期的原因？
5. 来来演练 —— 再来一次怎么做？
6. 下月规划 —— 如何调整目标和路径？

---

### 📈 年报生成

**触发方式**：
- "生成2026年年报"
- "帮我写年度复盘"
- "生成年报"

**包含四大维度**：
1. **热爱** —— 我喜欢什么？
2. **技能** —— 我擅长什么？
3. **金钱** —— 什么能变现？
4. **价值** —— 我能带来什么价值？

**六阶段评估**：
- 基础生存 → 自我认知 → 探索实践 → 能力跃迁 → 关系网络 → 系统超越

## ⚙️ 配置管理

### 查看当前配置

直接说："查看日志配置"

### 修改配置

| 想做的事 | 怎么说 |
|---------|--------|
| 改保存路径 | "把日志路径改成 D:\日记" |
| 改日期格式 | "改成 YYYY-MM-DD 格式" |
| 改提醒时间 | "提醒时间改成22:00" |

| 启用敏感词过滤 | "开启内容过滤" |

### 配置文件

配置文件位于：`[skill目录]/journal_config.json`

**完整配置示例**：
```json
{
  "root_path": "[你的日志保存路径]",
  "initialized": true,
  "preferences": {
    "date_format": "M.D",
    "auto_add_weather_weekday": true,
    "git_auto_commit": true
  },
  "safety": {
    "backup_before_overwrite": true,
    "backup_dir": ".backup",
    "config_protected": true,
    "show_path_before_delete": true,
    "confirm_before_delete": true,
    "force_git_protection": true
  },
  "reminders": {
    "daily_reminder": {
      "enabled": true,
      "time": "22:00",
      "message": "今天写日志了吗？"
    }
  }
}
```

## 🤔 常见问题

**Q: 没有安装Git能用吗？**
> 可以！Git自动提交是可选功能，没有安装时会自动跳过，不影响正常使用。

**Q: 日志保存在哪里？**
> 默认保存在你配置的 `root_path`，会在根目录下创建"年份/月份日志/"的文件夹结构。

**Q: 如何查看所有日志？**
> 说"打开总日志"，或者去保存路径找到"总日志.md"文件。

**Q: 能设置多个提醒吗？**
> 目前支持每日定时提醒 + 间隔提醒 + 月末提醒，后续会增强。
