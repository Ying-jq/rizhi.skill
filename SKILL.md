# 日志保存与分析 Skill

## 核心功能

本skill实现日志的保存、自动分析与周期性复盘生成功能。

## 配置文件

**配置文件**：`skill目录/journal_config.json`

**首次使用**：检查配置文件是否存在，如不存在则询问用户以下信息并创建配置文件。

### 首次使用引导

当检测到配置文件不存在时，询问用户以下内容：

```
🌸 欢迎使用日志保存与分析 Skill！

在使用之前，我需要帮你完成配置~

【必填】
1️⃣  日志保存路径
    请告诉我你希望保存日志的文件夹路径
    （例如：F:\日记、D:\我的日志 等）

【以下均为可选配置，可直接使用默认值，后续随时修改】

2️⃣  日期格式
    · M.D（如 5.18）
    · YYYY-MM-DD（如 2026-05-18）
    → 默认：M.D

3️⃣  自动添加星期和天气
    · 是 / 否
    → 默认：是

4️⃣  日志导出格式
    · md（推荐，支持富文本，与 Git 兼容性最好）
```

### 配置项说明

#### 1. 必填配置

| 字段 | 类型 | 说明 |
|------|------|------|
| `root_path` | string | **必填** 日志保存根目录路径 |

#### 2. preferences（用户偏好，可选）

| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `date_format` | string | `"M.D"` | 日期格式，可选 `"M.D"` 或 `"YYYY-MM-DD"` |
| `auto_add_weather_weekday` | boolean | `true` | 是否自动添加星期和天气信息 |
| `git_auto_commit` | boolean | `true` | 是否启用Git自动提交（无Git则自动跳过） |
| `export_format` | string | `"md"` | 日志导出文件格式，可选 `"md"` / `"txt"` / `"doc"` / `"pdf"` / `"rtf"` / `"html"` / `"odt"` |

**关于 `export_format` 的效率说明**：

| 格式 | 优点 | 缺点 | 推荐场景 |
|------|------|------|----------|
| `md`（推荐） | 支持标题、加粗、列表等富文本；与 Git 版本管理兼容性最好；文件体积小 | 需要支持 Markdown 的编辑器打开 | 日常使用、版本管理 |
| `txt` | 任何文本编辑器都能打开；兼容性最强；文件体积最小 | 无格式，纯文本；不便于阅读 | 纯存档、快速查看 |
| `doc` | 可直接用 Word 打开编辑；适合打印 | 生成效率最低；文件体积大；与 Git 兼容性差 | 需要 Word 协作时 |
| `pdf` | 版式固定，适合打印存档；任何设备打开效果一致 | 无法直接编辑；生成效率较低 | 正式存档、分享打印 |
| `rtf` | 支持基本格式（加粗、斜体、列表）；文件体积适中；与 Word 兼容 | 功能有限，不支持复杂排版 | 轻量格式兼容 |
| `html` | 可用浏览器直接打开；支持链接、图片等富媒体 | 默认无本地样式；需要额外处理样式 | 浏览器查看、需要内嵌网页 |
| `odt` | 开源格式，LibreOffice / WPS 可打开；支持基本格式 | 与 Word 兼容性一般 | 使用 LibreOffice 或 WPS 时 |

> 💡 建议选择默认的 `"md"` 格式，除非你有特定的格式需求。

#### 3. safety（安全保护，可选）

| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `backup_before_overwrite` | boolean | `true` | 覆盖已有日志前是否自动备份 |
| `backup_dir` | string | `".backup"` | 备份文件夹名称 |
| `config_protected` | boolean | `true` | 配置文件是否受保护（不可被普通操作删除） |
| `show_path_before_delete` | boolean | `true` | 删除操作前是否显示完整路径供确认 |
| `confirm_before_delete` | boolean | `true` | 删除操作前是否需要二次确认 |
| `force_git_protection` | boolean | `true` | 是否禁止强制git操作（如 --force） |

#### 4. content_filter（内容过滤，可选）

| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `enabled` | boolean | `false` | 是否启用敏感词过滤 |
| `replacement` | string | `"***"` | 敏感词的替换字符 |

#### 5. reminders（提醒功能，可选）

| 字段 | 子字段 | 类型 | 默认值 | 说明 |
|------|--------|------|--------|------|
| `daily_reminder.enabled` | - | boolean | `false` | 是否启用每日定时提醒 |
| `daily_reminder.time` | - | string | `"21:00"` | 提醒时间（24小时制） |
| `daily_reminder.message` | - | string | `"今天记得写日志哦~"` | 提醒消息 |
| `gap_reminder.enabled` | - | boolean | `false` | 是否启用间隔提醒 |
| `gap_reminder.threshold_days` | - | number | `3` | 超过N天没写日志时提醒 |
| `monthly_reminder.enabled` | - | boolean | `false` | 是否启用月末提醒 |
| `monthly_reminder.message` | - | string | `"本月最后一天，记得生成月志复盘哦~"` | 提醒消息 |
| `quiet_hours.enabled` | - | boolean | `false` | 是否启用免打扰时段 |
| `quiet_hours.start` | - | string | `"23:00"` | 免打扰开始时间 |
| `quiet_hours.end` | - | string | `"08:00"` | 免打扰结束时间 |

## 1. 日志保存功能

### 1.1 触发条件
当用户输入格式为"**日期+内容**"时（如"5.15 今天..."、"2026-05-15 今天..."），触发日志保存。

### 1.2 日期格式识别
支持以下日期格式：
- `M.D` 或 `M.DD`（如 `5.15`、`12.03`）→ 自动补充当前年份
- `YYYY.M.D` 或 `YYYY.M.DD`（如 `2026.5.15`、`2026.12.03`）→ 使用指定年份
- `YYYY-MM-DD` 或 `YYYY/M/DD` 等其他常见格式 → 提取年月日

### 1.3 文件夹结构
```
保存根目录/
├── 总日志.md                    # 汇总所有日志
├── 2025/
│   ├── 10月份日志/
│   ├── 11月份日志/
│   └── 12月份日志/
├── 2026/
│   ├── 1月份日志/
│   ├── 2月份日志/
│   ├── 3月份日志/
│   ├── 4月份日志/
│   └── 5月份日志/
│       └── 5.15.md
```

### 1.4 日期冲突检测与确认
**重要**：保存日志前必须检查该日是否已有日志文件。

1. 根据解析的日期，构造目标文件路径：`年份/X月份日志/M.D.md`
2. 检查该文件是否已存在
3. **如文件已存在**：
   - 询问用户：`检测到 {日期} 已存在日志，是否覆盖？`
   - 等待用户回复
   - 用户确认覆盖 → 覆盖保存
   - 用户拒绝覆盖 → 询问用户是否保存为新文件名（如 `M.D_2.md`）
   - 用户再次拒绝 → 取消保存操作
4. **如文件不存在**：直接保存

### 1.5 日志文件格式
**单个日志文件**（以日期命名，如 `5.15.md`）：
```markdown
# 5.15

[星期X，天气]

[用户原文内容]

---

## 分析

### 1.今天让我开心的
（分析原文，提取让用户开心的事）

### 2.今天让我充实的
（分析原文，提取让用户充实的事）

### 3.今天值得我感谢的
（分析原文，提取值得感恩的事）

### 4.我在今天的思考
（分析原文，进行深度思考）

### 5.今天我需要改进的
（分析原文，提取需要改进的地方）
```

### 1.6 总日志追加
将"**日期 + 原文 + 分析**"完整内容**追加**保存到根目录的"总日志.md"文件中。

**日期格式统一**：
- 如果原文日期带年份（如`2026.5.15`），直接使用
- 如果原文日期不带年份（如`5.15`），自动补充当前年份

**总日志格式**：
```markdown
# 2026-5.15

5.15
[星期X]，[天气]

[原文内容]

---

## 分析

### 1.今天让我开心的
...

### 2.今天让我充实的
...

### 3.今天值得我感谢的
...

### 4.我在今天的思考
...

### 5.今天我需要改进的
...

---
```

### 1.7 自动Git提交
每次保存日志后自动执行git提交：
```bash
git add .
git commit -m "auto: [日期时间] 保存日志"
```

```markdown
### 2. 自我认知
深入了解自己的天赋,能力,兴趣,习惯,金钱观等各项喜恶

做过很多事,收集很多信息,确认自己的目标

### 4. 能力发展
软能力,硬能力,实践经验

### 5. 校招技术
收集信息,简历,笔试,面试,谈判,决策等一系列素养

### 6. 系统思考
正确提问,复盘技术,自我评估,时间管理等一系列底层素养

---

### 当前可能存在的问题和劣势
（基于日志分析，指出具体问题和劣势）

（基于问题分析，提出下月需要重点关注的方面）
```

## 4. 日志搜索功能

### 4.1 触发条件
当用户输入以下类型指令时触发：
- "**搜索日志 [关键词]**"
- "**查找日志 [关键词]**"
- "**日志搜索 [关键词]**"
- "**在日志中找 [关键词]**"
- 或任何包含"搜索"、"查找"、"找"+"日志"的意图

### 4.2 搜索方式
使用 `scripts/search_logs.py` 脚本进行搜索。

**基本用法**：
```bash
cd "skill目录/scripts"
python search_logs.py -k "关键词"
```

**高级选项**：
- `-k, --keyword`: 搜索关键词
- `-s, --start`: 开始日期（格式: YYYY-MM-DD 或 M.D）
- `-e, --end`: 结束日期（格式: YYYY-MM-DD 或 M.D）
- `-t, --total`: 搜索总日志（默认开启）

**示例**：
```bash
# 搜索包含"朋友"的日志
python search_logs.py -k "朋友"

# 搜索2026年5月的日志
python search_logs.py -s 2026-05-01 -e 2026-05-31
```

## 5. 日志统计分析功能

- 总日志数
- 按月统计
- 按星期统计

#### 5.2.2 心情趋势分析

#### 5.2.3 高频关键词提取
- 提取日志中出现频率最高的关键词（排除停用词）
- 展示 TOP 10-20 关键词

### 5.3 使用方法

```bash
cd "skill目录/scripts"

```

- `-y, --year`: 分析指定年份（默认当前年）

## 6. 智能提醒功能

### 6.1 触发条件
当用户输入以下类型指令时触发：
- "**设置日志提醒**"
- "**提醒我写日志**"
- "**每天提醒我**"
- "**查看提醒设置**"
- "**取消提醒**"

### 6.2 提醒类型

#### 6.2.1 定时提醒
- **功能**：每天固定时间提醒用户写日志
- **实现方式**：
  - 方案A：使用 Windows 任务计划程序（推荐）
  - 方案B：使用 WorkBuddy 自动化功能（如支持）
- **可配置项**：
  - 提醒时间（如：每天21:00）
  - 提醒内容（如："今天记得写日志哦~"）
  - 提醒频率（每天/每周特定天）

#### 6.2.2 间隔提醒
- **功能**：如果超过N天没写日志，自动提醒
- **实现方式**：检查最近一篇日志的日期，计算间隔天数
- **可配置项**：
  - 提醒阈值（如：超过3天没写日志时提醒）
  - 提醒频率（每天检查一次/只在打开WorkBuddy时检查）

#### 6.2.3 重要日期提醒
- **功能**：在重要日期前提醒用户写总结
- **示例**：
  - 每月最后一天提醒写月志
  - 每年最后一天提醒写年报
  - 特定纪念日提醒

### 6.3 配置管理
提醒配置保存在 `journal_config.json` 的 `reminders` 字段中。

**配置示例**：
```json
{
  "root_path": "...",
  "initialized": true,
  "initialized_date": "...",
  "reminders": {
    "daily_reminder": {
      "enabled": true,
      "time": "23:30",
      "message": "今天记得写日志哦~"
    },
    "gap_reminder": {
      "enabled": true,
      "threshold_days": 3
    },
    "monthly_reminder": {
      "enabled": true,
      "day": "last",
      "message": "本月最后一天，记得生成月志复盘哦~"
    }
  }
}
```

### 6.4 实现方式

#### 方案A：Windows 任务计划程序
1. 创建 PowerShell 脚本 `send_reminder.ps1`
2. 使用 `schtasks` 命令创建定时任务
3. 在指定时间触发提醒

**创建定时任务示例**：
```powershell
# 创建每天21:00的提醒任务
schtasks /create /tn "WorkBuddy_日志提醒" /tr "powershell.exe -ExecutionPolicy Bypass -File send_reminder.ps1" /sc daily /st 21:00
```

#### 方案B：WorkBuddy 自动化（推荐）
如果使用 WorkBuddy 的自动化功能：
1. 读取配置文件中的提醒设置
2. 在每次会话开始时检查是否需要提醒
3. 如需提醒，向用户发送提醒消息

### 6.5 提醒检查逻辑
```
1. 检查今日是否已写日志
   - 是 → 不提醒
   - 否 → 继续检查

2. 检查是否启用定时提醒
   - 当前时间 >= 提醒时间 → 发送提醒
   - 当前时间 < 提醒时间 → 不提醒

3. 检查是否启用间隔提醒
   - 距离上次日志天数 >= 阈值 → 发送提醒
   - 距离上次日志天数 < 阈值 → 不提醒
```

### 6.6 未来增强
- 提醒消息个性化（根据用户心情、天气等）
- 与其他应用集成（如微信提醒、邮件提醒）
- 提醒效果跟踪（用户是否在提醒后写了日志）

## 7. 执行流程

### 7.1 日志保存流程
1. 读取配置文件获取保存路径
2. 如配置文件不存在，询问用户确认路径并创建配置文件
3. 解析日期 → 确定年份和月份
4. 检查并创建年份文件夹（如不存在）
5. 检查并创建月份文件夹（如不存在，格式：`X月份日志/`）
6. **检查目标日志文件是否已存在**
   - **如已存在**：询问用户是否覆盖，等待回复
     - 用户确认覆盖 → 覆盖保存
     - 用户拒绝覆盖 → 询问是否保存为新文件名（如 `M.D_2.md`）
     - 用户再次拒绝 → 取消保存，结束流程
   - **如不存在**：继续下一步
7. 撰写分析 → 保存日志文件（格式：`M.D.md` 或用户选择的新文件名）
8. 追加到总日志（**仅当用户选择覆盖或新建时追加**）
9. 自动git提交

### 7.2 月志生成流程
1. 读取配置文件获取保存路径
2. 解析月份 → 定位月份文件夹
3. 读取该月所有日志文件
4. 读取`references/月志复盘方法.md`获取方法论指导
5. 基于日志内容+方法论生成月志
6. 保存月志文件（命名：`X月日志.md`）
7. 自动git提交

### 7.3 年报生成流程
1. 读取配置文件获取保存路径
2. 解析年份 → 定位年份文件夹
3. 读取该年所有月份文件夹下的日志
4. 读取该年所有月志（如有）
5. 基于全年内容生成年报
6. 保存年报文件（命名：`X年年报.md`）
7. 自动git提交

### 7.4 日志搜索流程
1. 读取配置文件获取保存路径
2. 解析用户输入的搜索条件（关键词、日期范围）
3. 使用 `scripts/search_logs.py` 执行搜索
4. 解析并展示搜索结果
5. 如用户请求查看完整日志，读取对应文件

### 7.5 统计分析流程
1. 读取配置文件获取保存路径
2. 解析用户输入的分析要求（年份、分析维度）
3. 使用 `scripts/analyze_logs.py` 执行分析
4. 解析并展示分析结果
5. 可选：生成可视化图表

## 8. 安全保护措施

### 8.1 误操作防护

| 风险场景 | 防护措施 |
|----------|----------|
| 覆盖已有日志 | 必须用户确认，显示"检测到X月X日已有日志，是否覆盖？" |
| 删除日志文件 | 显示完整路径 + 二次确认 |
| 删除配置文件 | 配置文件受保护，普通操作不可删除 |
| 误读搜索结果 | 搜索结果显示日期+上下文片段 |
| 分析结果误导 | 所有分析结果标注"仅供参考，以原文为准" |

### 8.2 数据安全

| 措施 | 说明 |
|------|------|
| 覆盖前备份 | `safety.backup_before_overwrite=true` 时自动备份到 `.backup/` |
| Git保护 | `safety.force_git_protection=true` 时禁止 `git push --force` |
| 路径显示 | 删除/覆盖前显示完整路径供用户确认 |

### 8.3 内容安全

| 措施 | 说明 |
|------|------|
| 敏感词过滤 | `content_filter.enabled=true` 时启用 |
| 隐私提醒 | 分析/统计结果不涉及敏感个人信息 |

### 8.4 提醒安全

| 措施 | 说明 |
|------|------|
| 免打扰时段 | `quiet_hours.enabled=true` 时，在设定时段内不发送提醒 |
| 提醒频率控制 | 避免过度打扰用户 |

## 9. 注意事项

- **配置文件路径**：`skill目录/journal_config.json`，修改此文件即可自定义保存路径和提醒设置
- Git提交信息使用中文，格式：`auto: [日期时间] [操作描述]`
- 日志分析部分需要根据用户原文内容进行合理推断和分析
- 月志和年报的评估应基于实际日志内容，避免空洞的总结
- 如用户输入日期格式不明确，应询问用户确认
- 星期和天气可根据日期和实际情况推断
- 搜索和统计分析功能需要先确保Python环境已安装必要依赖（如无需额外依赖，使用标准库即可）
- 提醒功能如在Windows下使用任务计划程序，需确保PowerShell执行策略允许运行脚本
- 所有分析结果仅供参考，实际情况请以日志原文为准
